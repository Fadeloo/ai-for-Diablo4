# 数据来源与更新策略

## 结论

项目第一版以官方补丁页和官方资料片页锁定版本与职业事实，以社区机制资料辅助伤害模型，以社区 BD 页面回填具体配装。当前只把官方 3.1.0 补丁页可追溯的 278 条唯一装备固定词缀写入装备库种子；图标只保存第三方外部地址，不下载图片文件。完整装备库尚未写死，必须通过可审计导入器进入 `data/generated/` 或后续数据库。

## 来源优先级

1. 官方补丁、赛季公告、游戏内工具提示。
2. 有维护记录的社区数据库或开源工具。
3. 玩家测试与社区公式，仅可作为 `assumptions` 或 `needs_validation`。

## 后续全量装备导入

全量导入必须满足：

- 数据源 URL、版本、抓取时间、许可证或使用边界明确。
- 先导入 1-2 个装备样例并人工核对。
- 全量导入后运行 `npm run verify`。
- 生成文件放在 `data/generated/`，不要覆盖人工维护的领域模型。
- 真实物品图标只能在授权、许可明确或本地自制的前提下进入 `public/assets/`。
- 第三方图标可先以 `external_url_reference_only_no_asset_download` 形式接入，并保留本地图标回退。
