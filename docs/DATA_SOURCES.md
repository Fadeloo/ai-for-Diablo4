# 数据来源与更新策略

## 结论

项目第一版以官方补丁页锁定版本事实，以社区机制资料辅助伤害模型。完整装备库尚未写死，必须通过可审计导入器进入 `data/generated/` 或后续数据库。

## 来源优先级

1. 官方补丁、赛季公告、游戏内工具提示。
2. 有维护记录的社区数据库或开源工具。
3. 玩家测试与社区公式，仅可作为 `assumptions` 或 `needs_validation`。

## 后续全量装备导入

全量导入必须满足：

- 数据源 URL、版本、抓取时间、许可证或使用边界明确。
- 先导入 1-2 个装备样例并人工核对。
- 全量导入后运行 `npm run verify`。
- 生成文件放在 `data/generated/`，不要覆盖人工维护的领域模型。
